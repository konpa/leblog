Créer votre blog avec Jekyll

11 November 2015 • 8 mins read

fr | en
Dans ce tutoriel, nous allons créer un blog avec Jekyll, écrire notre premier article, ajouter une section commentaires avec Disqus et l'héberger gratuitement sur github.

Ce tutoriel fait parti d’une série de tutoriels sur comment créer, maintenir et personnaliser un blog avec Jekyll. Si vous n’avez pas lu les tutoriels précédents, allez y jeter un oeil:

  • Créer votre blog avec Jekyll
  • Maintenir son blog Jekyll efficacement avec Gulp
  • Personnaliser son blog Jekyll - Partie I
  • Personnaliser son blog Jekyll - Partie II

Introduction

Dans ce tutoriel, je vais vous montrer comment:

  • créer un blog avec Jekyll,
  • écrire notre permier article,
  • ajouter un espace de commentaire à notre article avec disqus,
  • et enfin, l’héberger gratuitement sur Github.

Vous pouvez voir et télécharger les fichiers sources de ce tutoriel sur son dépôt Github.

Créer un blog avec Jekyll

Dans un premier temps, il faut installer Jekyll sur notre ordinateur.

Maintenant que Jekyll est installé, nous pouvons générer notre blog.

Ensuite, déplacez-vous dans le nouveau dossier et lancez le serveur Jekyll.

Vous pouvez vous rendre sur http://127.0.0.1:4000/ et voir votre blog en action!

Il faut maintenant le personnaliser un peu. Toutes les configurations se trouvent dans le fichier _config.yml, situé à la racine de votre projet. Ouvrons-le dans notre éditeur favori pour l’éditer.

Nous allons supprimer la page “about” pour l’instant. Supprimer simplement le fichier about.md à la racine de votre projet.

Stoppez et relancez la commande jekyll serve et regardez le résultat!

C’est tout pour cette partie! Nous le garderons simple pour l’instant et ne personnaliserons pas plus notre blog. Mais restez attentif, un tutoriel expliquant comment faire ça arrivera bientôt ;)

Ecrire notre premier article

Premièrement, nous devons créer le fichier qui contiendra notre article dans le dossier _posts. Les fichiers article doivent être nommé suivant la convention YEAR-MONTH-DAY-title.MARKUP, nommez le donc comme suit 2015-11-18-our-first-awesome-post.md.

C’est une partie importante de votre article, elle ne sera pas visible dans votre article mais va permettre à Jekyll de générer votre article.

Après cela, vous pouvez commencer à écrire votre premier article en utilisant la syntaxe markdown. Je ne rentrerai pas les détails car un grand nombre de tutoriel/référence explique comment écrire en markdown.

Je vais juste expliquer comment gérer les images de votre article que vous voulez stocker localement. Pour insérer une image dans votre article, il faut écrire ![image title](http://image-url). Mais si votre image est stockée localement, il faudra utiliser un chemin relatif (ou absolu) de ce type ![image title](/img/image-name.jpg).

Nous allons donc créer un dossier img à la racine de notre projet qui contriendra toutes nos images. Ce dossier sera automatiquement copier dans le dossier _site qui contient notre site en production.

Maintenant vous pouvez ajouter une image contenu dans ce dossier comme ceci ![image title]({{ site.url }}/img/image-name.jpg)

Héberger notre site sur Github

Maintenant que nous avons ajouter du contenu à notre blog, nous voulons le mettre en ligne pour pouvoir le partager avec le monde entier.

Allez sur votre compte github et créer un nouveau dépôt (repository).

Et copiez le lien ssh (ou https) de votre dépôt.

Maintenant, ouvrez votre console à la racine de votre projet, allez dans le dossier _site et initialisez un nouveau dépôt git.

Ajouter le lien vers votre dépôt github que vous avez copié un peu plus tôt.

Mettez à jour l’adresse de votre site en production dans _config.yml.

Il faut maintenant créer la branche gh-pages et ensuite enregistrez le tout dans un commit.

Enfin, nous l’envoyons sur github.

Vous devriez maintenant voir votre site sur http://username.github.io/nom-du-dépôt!

C’est tout pour le moment… En attendant le prochain tutoriel!

Bonus: Redirigez votre nom de domaine vers votre blog

Si vous avez un nom de domaine, vous pouvez rediriger un sous domaine sur votre blog (ex: blog.votredomain.com).

Pour faire cela, vous devez commencer par créer un fichier CNAME à la racine de votre projet et y ajouter une ligne contenant l’adresse de redirection (ex: blog.yourdomain.com).

Enregistrez un commit et envoyez-le sur votre dépôt github.
